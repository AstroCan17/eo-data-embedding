# Design justification file

A standalone Design Justification File is **not applicable** to this project. For `eo-data-embedding`
— a single-developer, non-flight ground software project — the design rationale is folded directly
into the SDD ([§ Key design decisions](design): frozen encoder, exact FAISS `IndexFlatIP`, held-out
threshold methodology) and into the V&V report, with requirements-to-design traceability carried in
the [traceability matrix](traceability) and COTS reuse justified in the Software Reuse File (SRF).
A separate DJF is therefore tailored out (see SDP §3, DJF row); this page is retained only as a
pointer to those artefacts.
